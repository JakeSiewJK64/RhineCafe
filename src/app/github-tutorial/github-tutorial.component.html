Version Control with Github Desktop

Version control is a very important skill to have as a software developer since our softwares are constantly undergoing constant change. In order to keep track of our changes we need to use a method to keep track of them effectively. Github Desktop is a beginner friendly version control software in my opinion, in this blog I will demonstrate how version controlling with Github Desktop can be done! üëç

Step 1

Navigate to the repository's site and copy the URL. The URL can be found under the Code button.

Step 2

Navigate to Github Desktop, from there you can head over to the File tab and select Clone Repository or CTRL + SHIFT + O.

Step 3

A window will prompt for further setup when cloning the repository. Here, you can select URL and from there you can input the URL you just copied from the repository site and paste into the window. Local path can be anywhere as long as it is convenient for you.

Step 4

Congratulations! At this point, you have successfully cloned a repository. After that, you might want to create a separate branch in case you made an irreversible mistake or you want to make a significant change then creating a separate branch is encouraged in a project. To create a branch, navigate to Github Desktop > Current Branch > New Branch. Like this:

Step 5

In the preceeding window that prompts, you can choose either to create a branch based on the current condition of the main branch or the branch you are currently. This can depend on your project advisor or group leader.

Step 6

If you are satisfied with the changes, you can head back to Github Desktop and make a commit summary message. The description is optional but is benefitial for future references.

Conclusion

You are all set! If you made it this far, the rest of the project should be fine, hopefully. Just beware of merge conflicts. To avoid this, please consult with you advisor or group leader. To proceed, you might want to check out How to Create a Pull Request.
