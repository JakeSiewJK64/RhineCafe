Github Pull Requests

Github PR (Pull Request) is a way for project members to "Request" to have their changes merged with the main branch. A way to submit your "homework" or "task". Please note that contributions only count if you have:
  • Made a PR (Pull Request)
  • Starred a Repository
  • Forked a Repository
More on that here .

Step 1

After you have committed your changes to a branch, you will need to "push" your changes. *Take Note: If you have made a mistake in your commit before you pushed your changes, you can undo your commit and this will cancel the commit from being pushed to your branch.

Step 2

After pushing your changes, you will be asked to Create a Pull Request from Github Desktop or you can Compare and Pull Request from the Repository Site.

Step 3

Here, you can finalise your Pull Request title (mandatory) and message (if needed). Afterwards, click Create Pull Request to submit a pull request. Ask your supervisor or group leader for further instructions after that.

Conclusion

I hope the tutorial was helpful. It was intended for those who have not used Github as a Version Control method in their software developing journey. And, to save time explaining the basic concept on how to use it effectively. Happy Hacking!
