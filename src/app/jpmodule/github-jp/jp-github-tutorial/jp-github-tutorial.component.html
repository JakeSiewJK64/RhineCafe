Github Desktop 経てコードバージョンを管理

Github Desktop(ギートハブデスクトップ)はプログラミングコードの履歴とバージョンの管理システムGUIで現るソフトウェアです。 プロジェクトを行われ前に、ギートハブのリポジトリのセットアップできないなら困る。このブログでは、基本なセットアップを説明したいと思います。

第１つ目

まずは、リポジトリのサイトに行って、コードのURL（Codeのボタンで見つける）をコーピーして。

第２つ目

ギートハブデスクトップに戻って,ファイル(File) の設定に押して、Clone Repository（リポジトリをクーロン）を選ぶ。それとも CTRL + SHIFT + Oもできます.

第３つ目

Clone Repositoryを選んだ後、レポジトリの設定が現れる。その際に、第１つ目にコーピーしたリポジトリのURLを 「Repository URL」に入力して。「Clone」のボタンを押して前に ユーザーに探しやすいの為「Local Path」を確認するのは重要です。

大４つ目

このままよくできました！リポジトリをクローンするのは成功です。後は、念の為 Main Branch に何が困る戻れない場合を予防するとか大きなコードを変更があるとか 新しいブランチが作れる。ブランチを作る方法は、まずはギートハブデスクトップに戻れ、お好みブランチを選んで、「New Branch」を押して完成です。

第５つ目

ブランチの詳しい設定が現れた時、２つのブランチが現れる（今選んだブランチとMainブランチ）。お好みのブランチを選んで、そして「Create Branch」を押して完成です。

第６つめ

もし変更したコードに満足した時、変更の纏め（Summary Required）が書くことが必要です。変更のメッセージが空いても平気。そして「Commit to (ブランチの名称)」 を押して完成します。

纏め
