Cherry Picking with Github Desktop

Cherry Picking is a way for developers to "pick" a commit from a branch and transfer it to another branch. This is useful if a branch contains too many mistakes or there is a significant change which might jeopardise the current code base and needed to be transferred to a branch where the code is safe to use. In this blog, we will discuss how we can do that using Github Desktop. üëå

Step 1

Go to your Branch History > Right Click > Select Cherry Pick Commit. Like this:

Step 2

A window will prompt asking you to Select a Branch to cherry pick (transfer) the selected commit to.
*Take Note:
Should you have any existing changes in your current branch, you can stash your changes and come back to it later. Don't worry, they are not deleted.
