How to collaborate in a private Github Repository

Github has the option for you to make your repository private so that it remains hidden from other Github users. Attempting to enter the repository via URL will only result in a 404 request. However, in this blog we will not cover how to make your repository private but how to make your Private Github Repository visible to specific Github members and allow them to contribute to the private repository.

Step 1

Go to your private repository settings page. Settings can be found on the navigation tab of your repository. Like this:

Step 2

In the settings page, navigate to Manage access, from there you can click on "Invite a collaborator".

Step 3

A window will prompt asking you to enter the collaborator's name. If the collaborator exists on Github his/her name will show as a suggestion.

Step 4

If the process was followed correctly, a success message should appear indicating the collaborate has been sent the request to collaborate. Please note, that if the collaborator has not accepted the request via email he/she still does not have the permission to view the repository.

Collaborator's POV

Step 5

As the collaborator, head over to your email associated with your Github account. There should be an invitation to collaborate with the private repository. Click on View Invitation

Step 6

The link should direct you to Github where you can either decline or accept the invitation, click Accept Invitation.

Step 7

If you followed the steps correctly, a success message will appear and you will now be able to view and collaborate with the private repository.

*Special Note:

As the repository owner, you can revoke the collaborator's access to the repository if he/she is no longer a collaborator. Head over to the Manage Access Settings (as in Step 1), click on the Trash icon to remove the collaborator from the list.

Conclusion

It is important to keep sensitive repositories private and only invite trusted collaborators to your repositories especially if we are dealing with API keys or to avoid plagiarism in general in a university/college project. But above all, never share any code unless intended and you are responsible for what is shown in your repository.
