プルリクエスト

プルリクエストとは自分のコードをMainブランチとかお好みブランチに混ぜることにします。こういう行動のは自分の貢献を提出方法になります。 特定の行動がコントロリユーションとしてカウントされます：
  • プルリクエストできたこと
  • リポジトリをStarを付けたこと
  • リポジトリをフォークしたこと
詳しいくこちらへ： here .

第１つ目

コードの変更がコミットをできた後、コミットを「Push」すべきです。 *注意: もしコミットの中でミスが見つけたら、コミットを「Push」すること前に、ブランチの変更履歴でコミットを右クリックして、「Undo commit...」を選んで コミットを戻ることができます。

第2つ目

コミットを「Push」をした後、ギットハブデスクトップからCreate a Pull Request が選択できます あるいは リポジトリのサイトに Compare and Pull Request もできます.

第3つ目

このままプルリクエストのまとめとメッセージを編集できます。プルリクエストのメッセージは空いてもできますがまとめは空いてるはいけません。 プルリクエストのまとめとメッセージに満足したら、Create Pull Requestを押して、プルリクエストが完成です。 その後、上司と相談してできます。
